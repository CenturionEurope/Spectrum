In large organisations it may be important that one or multiple users are unable to change the permissions on their account.

Please note that once you have locked permissions you will be unable to change them via the Cole user interface and will have to unlock permissions to alter them again, so it's advised that you set the permissions of a user first before locking them.

You should only lock permissions for users if you are comfortable with MySQL® and should be preformed if you consider yourself a competent developer.

  1. Connect to the applicable database server using a MySQL® tool such as Sequel Pro and open Cole's database.
  2. Open the 
    Users
    table
  3. Find the user you want to lock the permissions of
  4. Edit the value of 
    PermissionsLocked
    from 0 to 1
  5. Save the changes to the database
  6. In Cole, click your profile picture on the Sidebar
  7. Click Edit on the applicable user that you altered the permissions for
  8. Ensure that the permissions matrix is locked.
